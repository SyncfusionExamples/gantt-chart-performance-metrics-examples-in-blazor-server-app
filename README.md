# Blazor Gantt Chart - Performance Metrics Examples

This repository contains a sample Blazor Server application that demonstrates how to analyze performance metrics related to the Syncfusion Blazor Gantt Chart. The sample focuses on measuring Gantt Chart loading time and tracking performance during common Gantt Chart operations.

## Project Overview

The Syncfusion Blazor Gantt Chart supports large data visualization and complex scheduling scenarios. Monitoring performance is essential when working with extensive task collections or advanced Gantt Chart features. This project showcases techniques to read and observe performance metrics such as initial load time and execution time for key Gantt actions.

The sample highlights how performance measurements can be collected during rendering, data processing, and interactive operations. These metrics help identify potential bottlenecks and assess the impact of configuration changes on overall Gantt Chart responsiveness in a Blazor Server environment.

## Features

- Measurement of Gantt Chart initial loading time
- Tracking performance of common Gantt Chart actions
- Evaluation of rendering behavior with data processing
- Integration with Syncfusion Blazor Gantt component
- Implementation in a Blazor Server application

## Prerequisites

- Visual Studio 2026
- .NET SDK with Blazor Server support
- Syncfusion Blazor Gantt NuGet package

## How to Run the Project

1. Clone or checkout this repository to a local folder.
2. Open the solution file in Visual Studio 2022.
3. Rebuild the solution to restore required NuGet packages.
4. Run the application and load the Gantt page.
5. Review the reported performance metrics during Gantt loading and actions.
