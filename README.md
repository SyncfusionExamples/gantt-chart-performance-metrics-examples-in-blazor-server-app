# Blazor Gantt Chart - Performance Metrics Examples

This repository contains a sample Blazor Web Application that demonstrates how to analyze performance metrics related to the Syncfusion Blazor Gantt Chart. The sample focuses on measuring Gantt Chart initial load time and tracking performance during common Gantt Chart operations.

## Project overview

The Blazor Gantt Chart is designed to handle **large data sets** and **complex scheduling scenarios**. When working with extensive task collections or advanced Gantt features, monitoring performance becomes essential to ensure a responsive user experience.

This project showcases techniques for capturing and analyzing performance metrics such as **initial render time, data processing duration**, and execution time for key Gantt Chart interactions. These measurements help identify potential performance bottlenecks and evaluate the impact of configuration or data changes on overall Gantt responsiveness.

The sample demonstrates performance monitoring in a server‑interactive Blazor environment, which is commonly used for enterprise‑scale Gantt applications.

## Features

- Measurement of Gantt Chart initial loading time
- Tracking performance of common Gantt Chart operations
- Evaluation of rendering behavior with data processing
- Performance analysis for large data scenarios
- Integration with Syncfusion Blazor Gantt Chart component

## Prerequisites

- Visual Studio 2022 (or later)
- .NET SDK 8.0 or later
- Syncfusion Blazor Gantt NuGet package
- A valid Syncfusion license (Community or Trial)

## How to Run the Project

1. Clone or checkout this repository to a local folder.
2. Open the project file (`.csproj`) in Visual Studio 2022 or later.
3. Restore the required NuGet packages.
4. Run the application.
5. Navigate to the page hosting the Gantt Chart.
5. Observe the reported performance metrics during initial load and while performing Gantt Chart actions.
